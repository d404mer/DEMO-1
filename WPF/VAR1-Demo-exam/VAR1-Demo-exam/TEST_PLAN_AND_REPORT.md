# План и отчет тестирования

## 1. Авторизация
- Ввод корректных `IdNumber` + `Password` -> переход в окно соответствующей роли.
- Неверный `IdNumber` (не число) -> предупреждение.
- Неверный пароль -> предупреждение.
- Пользователь без поддерживаемой роли -> сообщение об ошибке.

## 2. Ролевые окна
- Менеджер: открывается профиль, список пользователей, список заказов.
- Директор: открывается профиль, пользователи, заказы, блок отчетов.
- Производитель/Потребитель: отображается профиль и список заявок, доступно создание новой заявки.
- Перевозчик: отображается профиль и список заявок, доступно создание новой заявки.

## 3. Профиль пользователя
- Открытие формы редактирования профиля.
- Сохранение изменений.
- Повторное открытие окна роли с актуальными данными.

## 4. Работа с заявками
- Создание новой заявки с корректными данными.
- Проверка валидации поля `Дней в пути`.
- Обновление списка заказов после добавления.

## 5. Отчет по результатам
- Реализованная логика покрывает базовые позитивные и негативные сценарии.
- Все ключевые операции выполняются с обработкой исключений и сообщениями пользователю.
- Сборка через `dotnet build` в текущем окружении не завершилась из-за ограничения локального MSBuild task host (x86), что требует сборки через Visual Studio Build Tools/MSBuild из установленного VS.
